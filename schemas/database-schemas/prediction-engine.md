# Prediction Engine Database Schema

**Schema name:** `predictions`
**Owner service:** prediction-engine (Python/FastAPI)
**Storage type:** Standard PostgreSQL 16 tables

---

## Overview

The predictions schema stores ML model metadata, calibrated predictions, and the feature vectors used to generate each prediction. All tables use standard Postgres -- no TimescaleDB hypertables needed since query patterns are primarily key-based lookups rather than time-range scans.

Estimated volume: **100K-500K predictions/year** across all leagues and market types, growing at ~1,000-3,000 rows/day during active seasons.

---

## Schema Setup

```sql
CREATE SCHEMA IF NOT EXISTS predictions;
SET search_path TO predictions;
```

---

## Tables

### model_versions

Registry of trained ML model versions. Each row represents a specific trained model for a specific sport and market type. Only one model per sport/market_type combination may be active at a time.

```sql
CREATE TABLE predictions.model_versions (
    id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sport                sport_enum NOT NULL,
    model_type           market_type_enum NOT NULL,
    version              TEXT NOT NULL,
    algorithm            TEXT NOT NULL DEFAULT 'xgboost',
    trained_at           TIMESTAMPTZ NOT NULL,
    training_data_range  TSTZRANGE NOT NULL,
    training_samples     INTEGER NOT NULL,
    evaluation_metrics   JSONB NOT NULL DEFAULT '{}',
    feature_names        JSONB NOT NULL DEFAULT '[]',
    is_active            BOOLEAN NOT NULL DEFAULT FALSE,
    artifact_path        TEXT NOT NULL,
    notes                TEXT,
    created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE predictions.model_versions IS 'Trained model registry. ~50-100 rows over time, mostly historical.';
COMMENT ON COLUMN predictions.model_versions.sport IS 'Which sport this model is trained for (FOOTBALL, BASKETBALL, BASEBALL).';
COMMENT ON COLUMN predictions.model_versions.model_type IS 'Which market type this model handles (SPREAD, TOTAL, MONEYLINE, etc.).';
COMMENT ON COLUMN predictions.model_versions.version IS 'Semantic version or date tag (e.g., "v2.1", "2026-03-15").';
COMMENT ON COLUMN predictions.model_versions.algorithm IS 'ML algorithm used (e.g., "xgboost", "lightgbm").';
COMMENT ON COLUMN predictions.model_versions.training_data_range IS 'Time range of training data used [start, end).';
COMMENT ON COLUMN predictions.model_versions.evaluation_metrics IS 'Metrics on holdout set: {"brier_score": 0.21, "log_loss": 0.68, "calibration_error": 0.02, "accuracy": 0.54}.';
COMMENT ON COLUMN predictions.model_versions.feature_names IS 'Ordered list of feature names the model expects: ["home_rest_days", "away_injury_impact", ...].';
COMMENT ON COLUMN predictions.model_versions.artifact_path IS 'Path to serialized model file on mounted volume (e.g., "/models/football/spread/v2.1.xgb").';

-- Only one active model per sport+market_type
CREATE UNIQUE INDEX uq_model_versions_active
    ON predictions.model_versions (sport, model_type)
    WHERE is_active = TRUE;

-- Lookup by sport and market type
CREATE INDEX idx_model_versions_sport_type
    ON predictions.model_versions (sport, model_type, created_at DESC);
```

### predictions

Individual calibrated predictions generated by the ML models. Each row is a prediction for a specific game, market type, and selection. Immutable -- predictions are never updated.

```sql
CREATE TABLE predictions.predictions (
    id                     UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    game_external_id       TEXT NOT NULL,
    model_version_id       UUID NOT NULL REFERENCES predictions.model_versions(id),
    league                 league_enum NOT NULL,
    market_type            market_type_enum NOT NULL,
    selection              TEXT NOT NULL,
    predicted_probability  DECIMAL(6,5) NOT NULL,
    simulation_probability DECIMAL(6,5),
    implied_probability    DECIMAL(6,5),
    edge                   DECIMAL(6,5),
    confidence_lower       DECIMAL(6,5),
    confidence_upper       DECIMAL(6,5),
    feature_importance     JSONB NOT NULL DEFAULT '{}',
    created_at             TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT chk_predictions_probability_range
        CHECK (predicted_probability >= 0 AND predicted_probability <= 1),
    CONSTRAINT chk_predictions_confidence_range
        CHECK (confidence_lower >= 0 AND confidence_upper <= 1 AND confidence_lower <= confidence_upper)
);

COMMENT ON TABLE predictions.predictions IS 'Calibrated ML predictions. Immutable, append-only. ~100K-500K rows/year.';
COMMENT ON COLUMN predictions.predictions.game_external_id IS 'External game identifier, matched against lines-service and statistics-service.';
COMMENT ON COLUMN predictions.predictions.predicted_probability IS 'Final calibrated probability (0.0 to 1.0) after ML adjustment.';
COMMENT ON COLUMN predictions.predictions.simulation_probability IS 'Raw probability from simulation before ML adjustment. NULL if no simulation was run.';
COMMENT ON COLUMN predictions.predictions.implied_probability IS 'Market-implied probability from best available line at prediction time.';
COMMENT ON COLUMN predictions.predictions.edge IS 'predicted_probability - implied_probability. Positive = predicted edge over market.';
COMMENT ON COLUMN predictions.predictions.confidence_lower IS 'Lower bound of prediction confidence interval.';
COMMENT ON COLUMN predictions.predictions.confidence_upper IS 'Upper bound of prediction confidence interval.';
COMMENT ON COLUMN predictions.predictions.feature_importance IS 'Top features and their contribution: {"injury_impact": 0.15, "rest_days_diff": 0.08, ...}.';

-- Most recent prediction for a game/market (primary query pattern)
CREATE INDEX idx_predictions_game_market
    ON predictions.predictions (game_external_id, market_type, created_at DESC);

-- Model performance analysis
CREATE INDEX idx_predictions_model_version
    ON predictions.predictions (model_version_id, created_at DESC);

-- Time-range queries for retraining data
CREATE INDEX idx_predictions_created
    ON predictions.predictions (created_at DESC);

-- League-scoped queries
CREATE INDEX idx_predictions_league
    ON predictions.predictions (league, created_at DESC);
```

### feature_vectors

Input features used for each prediction, stored for explainability, debugging, and model retraining. One-to-one relationship with predictions.

```sql
CREATE TABLE predictions.feature_vectors (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    prediction_id   UUID NOT NULL UNIQUE REFERENCES predictions.predictions(id) ON DELETE CASCADE,
    features        JSONB NOT NULL,
    feature_sources JSONB,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

COMMENT ON TABLE predictions.feature_vectors IS 'Feature vectors for explainability and retraining. One per prediction. ~100K-500K rows/year.';
COMMENT ON COLUMN predictions.feature_vectors.features IS 'Feature name-to-value map: {"home_rest_days": 3, "away_injury_impact": -2.5, "line_movement_sharp": 1.2, ...}.';
COMMENT ON COLUMN predictions.feature_vectors.feature_sources IS 'Data freshness tracking: {"stats": "2026-03-29T18:00:00Z", "lines": "2026-03-30T10:15:00Z"}.';

-- Lookup by prediction
CREATE INDEX idx_feature_vectors_prediction
    ON predictions.feature_vectors (prediction_id);

-- Time-range queries for bulk retraining data export
CREATE INDEX idx_feature_vectors_created
    ON predictions.feature_vectors (created_at DESC);
```

## Example Queries

### Get the latest prediction for a game and market type

```sql
SELECT p.*, fv.features
FROM predictions.predictions p
LEFT JOIN predictions.feature_vectors fv ON fv.prediction_id = p.id
WHERE p.game_external_id = 'odds_api_abc123'
  AND p.market_type = 'SPREAD'
ORDER BY p.created_at DESC
LIMIT 1;
```

### Get active model for a sport and market

```sql
SELECT *
FROM predictions.model_versions
WHERE sport = 'FOOTBALL'
  AND model_type = 'SPREAD'
  AND is_active = TRUE;
```

### Export training data for a date range

```sql
SELECT p.game_external_id, p.league, p.market_type, p.selection,
       p.predicted_probability, p.simulation_probability,
       fv.features
FROM predictions.predictions p
JOIN predictions.feature_vectors fv ON fv.prediction_id = p.id
WHERE p.created_at BETWEEN '2025-09-01' AND '2026-02-01'
  AND p.league = 'NFL'
ORDER BY p.created_at;
```
